--- layout: docs-api title: Remove members from a group --- 
POST /db/{db}/groups/{id}/remove

Removes the provided people from the specified group. People not in the group (or in the database) are silently ignored.

Request

The request body should be an array of people identifiers.

Response

Always returns 202 Accepted: this is a delayed, idempotent request.

The response contains a single field:

at<clock>

The clock position when all the removed people will disappear in the group.

Examples

Example request

POST /db/0SNQc00211H/groups/0SNQe0032JZ/remove
Content-Type: application/json

[ "0SNQe00311H", "0SNQg00511H" ]

Example response

202 Accepted
Content-Type: application/json

{ "at": {"1":113} }

Errors

Returns 401 Unauthorized

  • if the provided token does not allow acting as {as}

Returns 403 Forbidden

  • if person {as} does not have at least moderate access to group {id}

Returns 404 Not Found

  • if database {db} does not exist
  • if group {id} does not exist in database {db}
  • if person {as} does not have at least view access to group {id}, to ensure absence equivalence

Access restrictions

The moderate access level is required to remove members from a group.
