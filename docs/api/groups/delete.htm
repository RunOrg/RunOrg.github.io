--- layout: docs-api title: Delete a group --- 
DELETE /db/{db}/groups/{id}

Deletes a group from the database forever. The people remain in the database, but their membership (to the deleted group) is forgotten.

Response

Always returns 202 Accepted: this is a delayed, idempotent request.

The response contains a single field:

at<clock>

The clock position when all the removed people will disappear in the group.

Examples

Example request

DELETE /db/0SNQc00211H/groups/0SNQe0032JZ
Content-Type: application/json

Example response

202 Accepted
Content-Type: application/json

{ "at": {"1":113} }

Errors

Returns 401 Unauthorized

  • if the provided token does not allow acting as {as}

Returns 403 Forbidden

  • if person {as} does not have at least admin access to group {id}
  • when attempting to delete group admin

Returns 404 Not Found

  • if database {db} does not exist
  • if group {id} does not exist in database {db}
  • if person {as} does not have at least view access to group {id}, to ensure absence equivalence

Access restrictions

The admin access level is required to delete a group. The admin group may not be deleted.
