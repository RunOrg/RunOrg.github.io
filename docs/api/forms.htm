--- layout: docs-api title: Forms ---

A form is a list of fields that can be filled by people. A single form can be filled several times, with each instance being tied to a separate owner (for instance, if a form is configured so that its owners are people, then each person can fill their own instance of the form).

A form contains one or more fields.

type <form.field>A field in a form

The following endpoints are used to create and manipulate the forms themselves.

type <form>An individual form and its fields
GET /db/{db}/forms/{id}/filledList all filled instances of a form
POST /db/{db}/formsCreate a form
GET /db/{db}/formsList all forms in the database
GET /db/{db}/forms/{id}Update form meta-data
GET /db/{db}/forms/{id}Read form meta-data

The following endpoints are intended to fill a form and retrieve the filled-in data.

PUT /db/{db}/forms/{id}/filled/{owner}Fill a form
GET /db/{db}/forms/{id}/filled/{owner}Read a filled form
