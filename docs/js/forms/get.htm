method RunOrg.Form.Load

If form is an instance of RunOrg.Form, then form.Load() fills (or replaces) its non-id fields with data retrieved from the API.

Returns a promise that will resolve to the form object if it could be filled.

Example usage

new RunOrg.Form("1ax69bfa335").Load().then(function(form) {     
  console.log("Fetched data for %s.", form.label)
})

Example usage, with error handling

var form = new RunOrg.Form("1ax69bfa335");

form.Load().then(function() {     
  console.log("Fetched data for %s.", form.label)
}, function(error) {
  if (error.HTTP == 404) 
    console.log("Form %s not found.", form.id)
  else 
    console.log("An error happened: %o", error)
})

Note: it is not necessary to Load() a form to call its other methods. You can, for example, run new RunOrg.Form(id).Stats() to get the fill statistics for a form without loading the form itself.

Errors

Returns 401 Unauthorized

  • if the provided token does not allow acting as {as}.

Returns 404 Not Found

  • if the database does not exist,
  • if form {id} does not exist in the database,
  • if person {as} does not have at least fill access to form {id}, to ensure absence equivalence.

Access restrictions

An access level of fill is required to view form information.
