As a server component

RunOrg sits next to your main database and helps your server-side code deal with the social aspects of your project.

As a web API

Your mobile application or JavaScript client can reach RunOrg over HTTPS. For simple projects, you don't even need a server.

We believe that social features should be a solved problem, not a wheel that every online project needs to reinvent.

Open Source

RunOrg is available from GitHub under the MIT license.

Our core principles:

  1. Do not create useless things.
    Resist the temptation to write features without a confirmed real-life use case, even if they are fun or trendy. Lines of code are a liability, not an asset.
  2. Data should never be leaked nor lost.
    Software should be secure and well-behaved, even if it takes longer to write. Some technology stacks are better than others for achieving this.
  3. Contributors should love the code.
    Have automated tests and clear documentation. Refactor mercilessly. Make it easy to run a local development version of RunOrg.
  4. Always follow standards when they exist.
    Standards get stronger the more they are used. They help newcomers learn faster. They make a lot of minor decisions so you don't have to.
  5. Developers should love the API.
    They deserve to be happy.
  6. Let the computer do the hard work.
    Humans create RunOrg. Type inference, static analysis, automated testing and other tools help us concentrate on the important parts.
  7. Improve speed and memory usage.
    When serving millions of API requests, milliseconds and kilobytes tend to add up.
  8. Be grateful for any outside help.
    It takes time and courage to offer a helping hand.

This not a bucket list, but a list of priorities: principles higher in the list take precedence over those below them.

Production-ready

RunOrg has been reviewed, tested and is already used in production. Download and install your own, or ask us for an instance on our own online production server at api.runorg.com !

We believe it is better to do things right from the start rather than coming back to fix them later on. There may be hundreds of features still on our road map, but all features we implemented so far are guaranteed to work and will keep your data safe and secure.
