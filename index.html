As a server component

RunOrg sits next to your main database and helps your server-side code deal with the social aspects of your project.

As a web API

Your mobile application or JavaScript client can reach RunOrg over HTTPS. For simple projects, you don't even need a server.

We believe that social features should be a solved problem, not a wheel that every online project needs to reinvent.

A successful web application is not about developing yet another social network from scratch. Your team should be solving hard problems, the ones that bring value to your users.

But what if you need user profiles or private messaging or file sharing or smart email notifications, tightly integrated with the rest of your application ?

You wouldn't write your own database engine, why should you rewrite standard social features yourself ?

RunOrg aims to solve those problems for you.
